# 03 · AI (Morning Brief + Takes)

The app has two AI outputs, both grounded **only** on the numeric market snapshot (the model has no
web access and is told never to invent figures):

- **Morning Brief** — a daily roll-up across all markets (lead paragraph, per-market lines, a hint).
- **Per-market Take** — a 1–2 sentence read shown in the detail panel.

Provider: **DeepSeek** (`deepseek-chat`) via its OpenAI-compatible chat API, JSON-object mode.

## Two ways to reach the model
Both implement the same `AiSource` interface, so the repository treats them identically. Which one
runs is decided by `DataPolicy.modeFor(DataSource.deepseek)`:

| Backend | File | When it's used |
|---------|------|----------------|
| **Direct** | `deepseek_source.dart` | Native (desktop/mobile) **with a `DEEPSEEK_API_KEY` in a local `.env`**. Calls DeepSeek directly. |
| **Proxy** | `ai_proxy_source.dart` | When `PROXY_URL` is set (and no local key). Calls the proxy, which holds the key. **Required on Web** (CORS + key). |
| **Mock** | `mock_source.dart` | Neither of the above → canned text. |

Precedence on native: a local `.env` key wins over the proxy. On Web only the proxy applies (no
filesystem). A `null` return from either backend means "use mock", which the repository honours — so
the app never breaks on the AI layer.

## Why the key is never in the app
An embedded API key can be extracted from a web bundle or a desktop/mobile binary. So:
- **Direct mode** reads the key from a `.env` **on disk at runtime** — it's never compiled in. On
  Web, a stub replaces `dart:io`, so the key can't even accidentally ship in the web bundle.
- **Proxy mode** keeps the key server-side entirely. This is the right choice for any distributed or
  web build.

See [05-running-and-config.md](05-running-and-config.md) for how to supply the key each way.

## Cost control (default)
Mirrors the original app: the **Morning Brief is live** (cached 24h), and per-market **Takes stay on
mock** to limit spend. Only the Brief actually calls the model on a normal run.

- Direct: flags `_liveBrief` / `_liveTakes` in `deepseek_source.dart`.
- Proxy: the `LIVE = { takes, brief }` object in `proxy/server.js`.

Flip `takes` on in whichever backend you use to enable live per-market Takes.

## Grounding & safety
- System prompt: use ONLY the snapshot numbers, no web access, no fabricated news/URLs/citations,
  informational (not advice), reply with a compact JSON object only.
- The snapshot handed to the model is `{ id, name, index, price, changePct, open }` per market.
- The Brief is cached for 24h and **only on a successful live call** — a transient failure doesn't
  lock the day to mock.
- The model sometimes returns empty/odd flags in Brief lines; the client overrides each line's flag
  from the market config by `id`, so the UI stays canonical.
